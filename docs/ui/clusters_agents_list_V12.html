1. Spec-Driven Development (SDD)

SDD ensures the Specification is the Product. All autonomous engineering stems from a versioned technical contract.

The Spec Index (spec_index.md): Central registry in Git pointing to all active specifications (product_reqs, architecture, tasks). Ensures agents and humans reference the same "Ground Truth."
Git-First Truth: Specs in Git enable versioning, branching, and a permanent audit trail. Engineering intent is never lost in ephemeral chat history.
Jira-Git Synergy: Every task in Jira is bi-directionally linked to a specific Git commit of SDD files for 100% traceability.
Deterministic Engineering: Agents execute against highly structured markdown, removing "intent guesswork" and ensuring measurable outcomes.

2. Engineering Guardrails

2.1 Cognitive Mode Separation (Mindset Specialization)

We never ask a single agent to "Understand, Plan, and Execute" simultaneously. Mindset isolation forces a system of checks and balances.

Surveyor vs Architect: One agent maps reality (What Is); the other proposes change (What Should Be).
Constructor vs Critic: One agent builds code; a separate entity with a fresh cognitive state verifies it.
ðŸš¨ Risk Prevented: Agentic Hallucination

Prevents an agent from "inventing" legacy system capabilities or functions to justify a flawed implementation plan. Cognitive Isolation grounds the Architect in verified facts.

2.2 Independent Reviewer Protocol

The reviewer must NEVER be the creator. We use heuristic diversity (different models, e.g., Claude reviewing GPT-4o) to catch logical gaps.

ðŸš¨ Risk Prevented: Confirmation Bias

Creators naturally view their implementation bugs as "intended behavior." Independent review provides the critical second opinion required for production-ready code.

2.3 Success-Contract First (TDD Safeguard)

Success contracts (Acceptance/Unit Tests) must be designed by a specialized UT Agent BEFORE code implementation begins.

ðŸš¨ Risk Prevented: Requirement Drift (Scope Creep)

Agents adding "nice-to-have" code or drifting from the PRD. The Binary Boundary of TDD forbids the agent from leaving the specification box.

2.4 Targeted Context Injection

Precision context packets via AST scanning Repo Mapper rather than full file dumps.

ðŸš¨ Risk Prevented: Context Window Dilution

Ignoring critical constraints due to noise (Lost-in-the-Middle). Precision injection keeps reasoning power focused on the specific logic delta.

2.5 Micro-Task Slicing (Atomic Boundaries)

Requirements are sliced into technical tasks taking < 3 iterations. Small tasks prevent state drift.

ðŸš¨ Risk Prevented: Agentic Debt & State Drift

Agents losing track of goals in long reasoning chains. Atomic logic blocks ensure focus remains 100% on the atomic sub-task spec.

3. Full Cluster & Agent deep dive

Cluster 0: Governance (Strategic Brain)

  • Manager Agent: The exclusive commit gateway. Orchestrates cluster transitions and manages the HITL queue.
  • Repo Mapper Agent: Injects AST-based context packs. Ensures zero-hallucination engineering by mapping global dependencies.
  • Data Insight Agent: Monitors "Human Bottlenecks" and agentic failure rates to optimize sprint velocity.

Cluster 1: Discovery (Requirements)

  • PRD Agent: Slices high-level intents into atomic, independent User Stories.
  • UI/UX Agent: Generates visual prototypes (mocks) to align human expectations early.
  • Acceptance Agent: Defines "Success Contracts" in ATDD/Gherkin format to ensure value delivery.

Cluster 2: Design & Planning

  • Arch Agent (Surveyor): Analytical map of "What Is" (Legacy/API). Prevents Legacy Corruption.
  • Solution Agent (Architect): Creative planning of "What Should Be" using ONLY Surveyor-verified interfaces.
  • Design Gatekeeper: Automated auditor ensuring solution adheres to Arch Layer 1 and organizational policy.

Cluster 3: Development (TDD Engine)

  • Planner Agent (Decomposer): Converts designs into technical sub-tasks (`tasks.md`).
  • UTest Agent: Sets "Goal Posts" before coding. Prevents Tautological Testing.
  • Coding Agent: Contract fulfiller; implements logic to pass binary test status.
  • Debugger Agent: Triggered on Fail > 4 Rule. Analyzes stack traces to provide logic patches.
  • Automation Agent: Strategic quality; identifies related regression flows & progression scripts.
  • Reviewer Agent: Independent gate (separate model) enforcing Clean Code and security.

Cluster 4 & 5: Validation & Deployment

  • Validation Agent: Full integration testing (Playwright) in isolated Staging.
  • Security Agent: SAST/SCA auditing. Autonomous speed must not sacrifice safety.
  • Release Mgmt Agent: Packages artifacts and populates notes/changelogs.
  • Deployment Agent: CI/CD orchestrator; marks epics as "Live" in Jira.
  • Env Monitoring: Post-live health checks and verification of "Live Hashes."
